# ドキュメント作成規約

`doc/` ディレクトリ配下の技術文書を作成・更新する際の規約を定める。

## 基本原則

- **言語**: 日本語で記述する（技術用語は英語のままで良いが、説明文は日本語）
- **形式**: Markdown（`.md`）
- **文字エンコーディング**: UTF-8
- **改行コード**: LF

## ファイル命名規則

- 小文字スネークケース（`snake_case`）
- 拡張子は `.md`
- 例: `kicl-web.md`, `database.md`, `processing-flows.md`

## ドキュメント構成

各ドキュメントは以下の標準セクション構成に従う。対象に応じて省略可。

```markdown
# タイトル: 副題（任意）

## 概要
ドキュメントの目的と対象読者を1-2行で簡潔に説明。

## アーキテクチャ
システムの全体構成や関係性を記述。

### 全体構成
図解やツリー構造で表現。

### 技術スタック
| 層 | 技術 | 用途 |
|---|---|---|

## ディレクトリ構造
```
module/
├── src/
│   └── commonMain/
└── build.gradle.kts
```

## 依存関係
外部依存やモジュール間の依存を記述。

## ビルドと開発
開発手順、前提条件、コマンド例を記述。

## CI/CD
ワークフロー、トリガー、デプロイ手順を記述。

## 機能一覧
モジュールが提供する機能を列挙。

### ルーティング
| パス | コンポーネント | 説明 |
|------|---------------|------|

## 処理フロー
テキストベースの図解で表現。

## 既存の◯◯との比較
| 項目 | 従来 | 新規 |
|---|---|---|

## 今後の展望
実装予定、課題、検討事項を列挙。
```

## 見出しレベル

| レベル | 用途 |
|--------|------|
| `#` | ドキュメントタイトル（1つのみ） |
| `##` | 大セクション（概要、アーキテクチャ、機能一覧など） |
| `###` | 中セクション（全体構成、技術スタック、個別機能など） |
| `####` | 小セクション（設定項目、型定義など） |

## コードブロック

- 言語指定を必ず行う（`bash`, `kotlin`, `typescript`, `json`, `yaml`, `sql` など）
- シェルコマンドには `$` プレフィックスを付けない（コピー＆ペーストしやすくするため）
- 長いコマンドはバックスラッシュで改行

````markdown
```bash
./gradlew :kicl:kicl-domain:jsBrowserProductionLibraryDistribution \
           :kicl:kicl-usecase:jsBrowserProductionLibraryDistribution
```
````

```kotlin
@JsExport
object KiclDomain {
    const val VERSION = "0.0.1"
}
```

## テーブル

- Markdown テーブル形式を使用
- 1列目に見出し・項目名、2列目以降に説明や値を配置
- 列幅は内容に合わせて適切に調整（無理に揃えない）

## 図解（ASCII アート）

- 処理フローはテキストベースの図解で表現
- インデントはスペース4
- 矢印は `→`, `↓`, `├─`, `└─` を使用
- 条件分岐やエラーパスも明確に示す

```
WebSocket フレーム受信
  │
  ▼
処理A
  ├─ 正常 → 結果B
  └─ エラー → エラーハンドリング
```

## リンク

- 相対リンクを使用（`../` や同階層のファイル名）
- 外部リンクは完全なURLを記載
- リンクテキストは簡潔に

## インラインコード

- ファイル名、ディレクトリ名、コマンド、変数名、型名はバッククォートで囲む
- 例: `KiclDomain`, `routes.tsx`, `npm run build`

## ドキュメントの更新ルール

1. **新規作成**: 新機能・モジュール追加時に対応するドキュメントを作成
2. **既存更新**: 機能変更・追加時は関連ドキュメントも同時に更新
3. **README更新**: 新ドキュメント作成時は `doc/README.md` のドキュメント一覧にも追加
4. **PR作成**: ドキュメントのみの変更でもPRを作成し、レビューを受ける

## 既存ドキュメントとの関係

- `doc/README.md` — ドキュメント全体のインデックス。新文書作成時は必ず更新
- `AGENTS.md` — AIエージェント向け開発ガイド。コードスタイル・コマンドを記載
- 各モジュール固有の詳細は `doc/` 配下の個別ファイルに記載
